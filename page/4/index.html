Killing process with one command, which is still utilizing the port

Jul 28, 2020
Killing the process It happens regularly that when you stop the process(in my case mostly a node server or flask server) and restart in the same terminal session. It gives the error saying PORT is still in use. In that case, using ps aux and grep, you find the process id, and kill it. In this stackoverflow answer by Hamid Tavakoli. This simple one line does it for you 1 kill -9 $(ps aux | grep '\snode\s' | awk '{print $2}') the above serves as an example for killing an node js process, which is still utilizing the port.…
